# 属性

  • 不能简写

    • UILabel --- xxLabel，不能 xxLb、xxLab 等

    • UIButton --- xxButton，不能 xxBt、xxBtn 等

    • UIImageView --- xxImageView，不能 xxImageV、xxImgV 等

    • NSDictionary --- xxDictionary，不能 xxDict、xxDic 等

    • NSMutableDictionary --- xxMDictionary，不能 xxDict、xxDic、xxMDict、xxMDic 等

    • NSArray --- xxArray，不能 xxs、xxArr 等

    • NSMutableArray --- xxArray，不能 xxs、xxArr、xxMArr 等

    • UIImage --- xxImage，不能 xxImg 等

  • 特殊约定

    • params : 参数字典
    • datas：数据源数组

    ❌错误写法❌：

    @property (nonatomic, strong) UIImage *productImg;

@property (nonatomic, copy) NSDictionary *skuDic;

@property (nonatomic, copy) NSArray<NSString *> *selectedFilterIds;

    ✅正确写法✅：

    @property (nonatomic, strong) UIImage *productImage;

@property (nonatomic, copy) NSDictionary *skuDictionary;

@property (nonatomic, copy) NSArray<NSString *> *selectedFilterIdArray;

  • 注意空格

    ❌错误写法❌：

    @property(nonatomic, strong) UIImage *productImage;
@property (nonatomic,strong) UIImage *productImage;
@property (nonatomic, strong)UIImage *productImage;
@property (nonatomic, strong) UIImage*productImage;
@property (nonatomic, strong) UIImage * productImage;
@property (nonatomic, strong) UIImage* productImage;

    ✅正确做法✅，参考上面的。

  • 关键字顺序

nonatomic ---> strong / weak / copy / assign ---> readonly / readwrite ---> nullable / nonnull

❌错误写法❌：

@property (strong, nonatomic) UIImage *productImage;
@property (nonatomic, readonly, strong)UIImage *productImage;

✅正确做法✅，参考上面的。

# 注释

为了更规范，推荐大家使用 CodeSnippets。

  • 示例 1 ❌
@property (nonatomic, copy) NSString *origin; //来源，打点使用的参数

  • 示例 2 ✅
/// 来源，打点使用的参数
@property(nonatomic, copy) NSString *origin;

下面这种注释效果等同于示例 2，可以这样写，但不推荐：

/// 备注：后来经过讨论，这种项目不可用
/** 来源，打点使用的参数 */
@property(nonatomic, copy) NSString *origin;

最后，看下 Xcode 快捷注释，如下图所示：

  • Xcode 自带注释快捷键： command + option + /
  • 如果上面快捷键失效，手动手写，请注意 ** /// 和 Description 之间有空格 **

# 代理

讨论后更新，代理规范：注意不要方法名字过长。

如 - (void)categoryIconViewDidScrollToPrevious:(VVCategoryIconView *)categoryIconView; 可更改为 - (void)iconViewDidScrollToPrevious:(VVCategoryIconView *)iconView; ，因为 VVCategoryIconView 我们已经知道是分类用的。

  • 错误示例❌

    @protocol VVCategoryIconViewDelegate <NSObject>

/**
 点击某个icon

 @param index 索引
 */
- (void)selectItemAtIndex:(NSInteger)index;

@optional

/**
 滑动到下一个tab，这里是为首页分类写的代理方法，当滑动collectionView超过最大偏移量的时候需要调用该方法,选中下一个tab
 */
- (void)scrollToNext;
/**
 滑动到上一个tab, 这个也是为首页分类写的代理方法，当滑动collecionView的偏移量小于最小偏移量的时候，需要选中上一个一个tab
 */
- (void)scrollToPrevious;

@end

  • 正确示例✅

    @protocol VVCategoryIconViewDelegate <NSObject>

@optional
- (void)categoryIconView:(VVCategoryIconView *)categoryIconView didSelectItemAtIndex:(NSUInteger)index;

/// 滑动到下一个tab，这里是为首页分类写的代理方法，当滑动collectionView超过最大偏移量的时候需要调用该方法,选中下一个tab
- (void)categoryIconViewDidScrollToNext:(VVCategoryIconView *)categoryIconView;

/// 滑动到上一个tab, 这个也是为首页分类写的代理方法，当滑动collecionView的偏移量小于最小偏移量的时候，需要选中上一个tab
- (void)categoryIconViewDidScrollToPrevious:(VVCategoryIconView *)categoryIconView;

@end

# 代理（错误做法 VS 正确做法）

参考 UITableViewDataSource / UITableViewDelegate ，思考一下。

  • selectItemAtIndex: scrollToNext 以及 scrollToPrevious 在 xxViewController （代理实现者）里，错误做法不能一眼看出是谁的代理，而正确做法可以；

  • 如果项目中有不同的代理，但是代理的方法相同，假如 xxViewController （代理实现者）同时遵循这 2 个协议，那么怎么办？很明显，正确做法可以根据 categoryIconView 判断，而错误做法无法判断。
